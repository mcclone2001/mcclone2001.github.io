Docker Cheatsheet

March 10, 2019

Construir una imagen

--build-arg <nombre argumento>=<valor> Para pasar parametros al Dockerfile, únicamente estarán disponibles en la instrucción docker build, si deseas que los parametros esten disponibles durnate la instruccion docker run ó docker start debes asignar el build-arg a una variable de entorno ENV <nombre variable de entorno>=${<nombre argumento>}

docker build <Ubicación del archivo Dockerfile> -t <nombre de imagen>

Instalar un contenedor

--name Para indicar el nombre del contenedor

-d Para dejar corriendo el contenedor en segundo plano

-it para disponer de entrada estandar y TTY

-v <ruta local>:<ruta en el contenedor> para compartir una carpeta entre el host y el contenedor, cuando usamos Docker Toolbox se espera que la ruta local este dentro de C:\Users y que el antivirus no escanee el volumen compartido, amdos son requisitos para evitar problemas de permisos.

-p <puerto local>:<puerto en el contenedor> para redirigir el tráfico de un puerto en el host al contenedor

--rm elimina automaticamente el contenedor en cuanto se cierra

--entrypoint=<comando> ejecuta el comando e ignora cualquier instrucción CMD del Dockerfile

--name <nombre> asigna un nombre al contenedor, en lugar de dejar que se genere aleatoriamente

docker run -dit <nombre de imagen>

docker run -dit -v /c/Users/USUARIO/Desktop/clojure/alpha/home:/tmp/proyecto clojure:dev

docker run -dit --rm --name clojure --entrypoint=/bin/bash -v /c/Users/USUARIO/Desktop/clojure/alpha/home:/tmp/proyecto clojure:dev

Listar todos los contenedores

docker ps -a

Detener contenedor

docker stop <nombre del contenedor>

Iniciar contenedor

docker start <nombre del contenedor>

Conectarse a la terminal del contenedor

El contenedor debe haberse instalado con los parametros -dit

docker attach <nombre del contenedor>

Desconectar la terminal del contenedor

ctrl+p ctrl+q

Crear una nueva terminal para conectarse al contenedor

https://stackoverflow.com/questions/19688314/how-do-you-attach-and-detach-from-dockers-process/30907056

sudo docker exec -ti [CONTAINER-ID] bash

Listar todas las imagenes disponibles

docker images -a

Eliminar imagen

docker rmi <nombre de la imagen>
